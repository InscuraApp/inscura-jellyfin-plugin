# Inscura용 Jellyfin 메타데이터 플러그인

언어: [English](../README.md) | [简体中文](README-zh.md) | [日本語](README-ja.md) | **한국어**

Inscura는 영화 정보, 출연진, 장르, 커버, 배경 이미지, 예고편 등의 자료를 정리할 수 있는 로컬 미디어 라이브러리 앱입니다. Jellyfin은 미디어 라이브러리의 재생 및 관리를 담당하지만, Inscura에서 이미 정리해 둔 이러한 데이터에 대해서는 알지 못합니다.

이 플러그인은 Inscura의 로컬 API 서비스를 Jellyfin에 연동하는 데 사용됩니다. Jellyfin이 영화 메타데이터를 새로 고칠 때, 플러그인은 제목, 소개, 개봉일, 평점, 장르, 제작사, 출연진, 커버, 배경 이미지 및 썸네일 등의 데이터를 Jellyfin에 기록합니다.

이 플러그인은 Inscura의 미디어 라이브러리 데이터와 라이브러리 내에서 생성된 이미지 리소스만 읽을 뿐, 원본 미디어 파일을 다운로드, 이동, 이름 변경, 삭제 또는 수정하지 않습니다.

## 현재 기능

- Jellyfin에서 전달된 실제 파일 경로와 파일명을 우선적으로 매칭하여, Jellyfin 제목이 수동으로 수정되어 발생하는 오매칭을 줄입니다.
- 경로 매칭이 불가능할 경우, 파일명 내 식별 코드와 Jellyfin 제목을 계속해서 시도합니다.
- 제목, 원제, 소개, 출시일, 연도, 평점, 장르, 제작사, 국가, 태그, 출연진, 감독, 각본가, 프로듀서, 포스터, 배경 이미지 및 썸네일을 포함하여 Jellyfin이 수용할 수 있는 영화 메타데이터 필드 작성을 지원합니다.
- Inscura에서 로컬 이미지 리소스를 가져와 Jellyfin의 Primary, Backdrop, Thumb, Banner, Logo, Art 및 Disc 이미지 후보로 사용할 수 있습니다.
- 원격 예고편은 현재 YouTube 주소만 가져옵니다. **YouTube가 아닌 로컬 또는 온라인 예고편은 Inscura 앱의 NFO 기능을 통해 내보낸 후 Jellyfin에서 스캔하도록 하십시오.**

## Inscura 로컬 인터페이스 서비스 활성화

1. Inscura를 실행하고 Jellyfin과 동기화할 미디어 라이브러리를 엽니다.
2. 설정 메뉴의 API 서비스 설정으로 이동하여 로컬 API 서비스를 활성화합니다.
3. API 서비스가 토큰 인증을 사용하는 경우, 설정 페이지에 표시된 API 토큰을 저장하십시오.
4. Jellyfin 서버가 설치된 기기에서 상태 확인 주소에 접속하여 서비스에 접근 가능한지 확인합니다.

예시:

```bash
curl "http://[ip]:28687/api/v1/health"
```

Jellyfin과 Inscura가 같은 컴퓨터에 설치되어 있지 않은 경우, 플러그인의 서비스 주소란에 `127.0.0.1`을 입력해서는 안 되며, Inscura가 실행 중인 컴퓨터의 LAN 내 주소를 입력해야 합니다. 예:

```text
http://[ip]:28687
```

로컬 API 서비스는 현재 미디어 라이브러리의 수명 주기에 따라 실행됩니다. 미디어 라이브러리가 열려 있고 서비스가 활성화된 상태에서는 포트를 감시하며, **미디어 라이브러리가 잠기거나 닫히거나 애플리케이션이 종료되면 서비스가 중지됩니다.**

## 권장 설치 방법: 플러그인 저장소

이 방법은 Jellyfin이 플러그인 저장소의 `manifest.json`을 읽어오므로, 설치 및 후속 업그레이드가 더 편리합니다. 중국 본토의 네트워크 환경에서는 jsDelivr 주소를 사용하는 것이 좋습니다.

저장소 이름:

```text
Inscura
```

저장소 URL:

```text
https://cdn.jsdelivr.net/gh/InscuraApp/inscura-jellyfin-plugin@release/manifest.json
```

설치 단계:

1. Jellyfin 콘솔로 이동합니다.
2. 플러그인 디렉터리 또는 플러그인 설정 페이지를 엽니다.
3. 저장소(Repositories)로 이동합니다.
4. ‘추가’를 클릭합니다.
5. 이름에 `Inscura`을 입력합니다.
6. URL에 위의 jsDelivr 매니페스트 주소를 입력합니다.
7. 저장한 후 플러그인 목록 또는 Catalog로 이동합니다.
8. `Inscura`을 찾아 설치합니다.
9. Jellyfin을 재시작합니다.

GitHub에 안정적으로 접속할 수 있는 경우, release 브랜치의 매니페스트 파일을 사용할 수도 있습니다. 단, 중국 본토 환경에서는 위의 jsDelivr 주소를 우선적으로 사용하십시오.

## 대체 설치 방법: zip 파일 직접 다운로드

Jellyfin에서 플러그인 저장소를 통해 설치할 수 없는 경우, 플러그인 zip 파일을 수동으로 다운로드하여 Jellyfin 플러그인 디렉터리에 넣을 수 있습니다.

### 다운로드 주소 확인

manifest 파일을 엽니다:

```text
https://cdn.jsdelivr.net/gh/InscuraApp/inscura-jellyfin-plugin@release/manifest.json
```

그중 최신 버전인 `sourceUrl`을 복사한 후, 해당 zip 파일을 다운로드합니다.

현재 공개된 파일의 주소 형식은 다음과 같습니다:

```text
https://cdn.jsdelivr.net/gh/InscuraApp/inscura-jellyfin-plugin@release/releases/Inscura_[버전].zip
```

### 플러그인 파일 배치

Jellyfin의 데이터 디렉터리를 찾아, 그 안의 `plugins` 디렉터리로 이동합니다. 일반적인 위치는 다음과 같습니다. 시스템이 이와 다를 경우, Jellyfin 콘솔에 표시된 데이터 디렉터리 또는 배포 시 실제 매핑된 경로를 기준으로 하십시오.

| 환경 | 일반적인 플러그인 디렉터리 |
| --- | --- |
| Linux 패키지 설치 | `/var/lib/jellyfin/plugins/` |
| Docker | 호스트가 컨테이너의 `/config/plugins/` 디렉터리로 매핑된 위치 |
| Windows 서비스 설치 | `%ProgramData%\Jellyfin\Server\plugins\` |
| Windows 포터블 또는 사용자 모드 | Jellyfin 데이터 디렉터리 내의 `plugins\` |

수동 설치 단계:

1. Jellyfin을 중지하거나, 복사가 완료된 후 Jellyfin을 재시작할 준비를 하십시오.
2. `plugins` 디렉터리 아래에 버전 디렉터리를 생성합니다. 예:

```text
plugins/Inscura_0.1.0.0/
```

3. zip 파일을 압축 해제하고, 그 안의 파일을 해당 디렉터리에 넣습니다.
4. 디렉터리에는 플러그인 DLL만 있으면 됩니다:

```text
Jellyfin.Plugin.Inscura.dll
```

5. Jellyfin 프로세스가 해당 파일에 대한 읽기 권한을 가지고 있는지 확인하십시오.
6. Jellyfin을 재시작하십시오.
7. 콘솔의 플러그인 페이지로 이동하여 `Inscura`이 로드되었는지 확인하십시오.

## Jellyfin에서 플러그인 활성화

플러그인을 설치한 후, 영화 라이브러리에서 Inscura 메타데이터 소스를 활성화해야 합니다.

1. Jellyfin 콘솔로 이동합니다.
2. 미디어 라이브러리를 열고 영화 라이브러리를 편집합니다.
3. 메타데이터 다운로더에서 `Inscura`을 활성화합니다.
4. `Inscura`을 다른 영화 메타데이터 소스보다 앞에 배치하는 것이 좋습니다.
5. 이미지 다운로더에서 `Inscura`을 활성화합니다.
6. 미디어 라이브러리 설정을 저장합니다.
7. 플러그인 설정 페이지로 이동하여 Inscura API 주소와 API 토큰을 입력합니다.
8. 영화에 대해 메타데이터 새로 고침 또는 식별을 실행합니다.

처음 사용할 때는 소수의 영화만 선택하여 메타데이터를 새로 고친 후, 제목, 출연진, 표지 및 배경 이미지가 예상대로 표시되는지 확인한 다음, 데이터베이스를 일괄적으로 새로 고치는 것이 좋습니다.

## 플러그인 설정 설명

| 설정 | 설명 |
| --- | --- |
| Inscura API URL | Inscura 로컬 API 서비스 주소. Jellyfin과 Inscura가 같은 서버에 있지 않을 경우 반드시 LAN 주소를 입력해야 합니다 |
| API 토큰 | 로컬 API 서비스에서 토큰 인증을 사용할 경우 입력합니다. Inscura가 인증 없음으로 설정된 경우 비워둘 수 있습니다 |
| 검색 결과 제한 | 매칭 시마다 Inscura에서 요청하는 후보 수 |
| 요청 타임아웃 | Inscura 로컬 API에 대한 요청 타임아웃 시간 |
| 영화 메타데이터 제공자 활성화 | 영화 메타데이터 스크래핑을 활성화하거나 비활성화합니다 |
| 이미지 제공자 활성화 | 이미지 가져오기를 활성화하거나 비활성화합니다 |
| YouTube 예고편 가져오기 | 활성화하면 YouTube 원격 예고편만 가져옵니다 |
| Inscura 미리보기 이미지를 썸네일 후보로 사용 | Inscura 미리보기 이미지를 Jellyfin 썸네일 후보로 사용 |
| 갤러리 이미지를 배경 이미지 후보로 사용 | Inscura 갤러리 사진을 Jellyfin 배경 이미지 후보로 사용 |

## 사용 권장 사항

- 처음 사용할 때는 소량의 영화만 먼저 새로 고침하여, 매칭 결과가 예상대로인지 확인한 후 일괄 새로 고침을 진행하십시오.
- Jellyfin에서 영화 제목이 수동으로 변경된 적이 있더라도, 플러그인은 제목을 유일한 기준으로 삼지 않고 실제 파일 경로와 파일명을 우선적으로 사용하여 매칭합니다.
- Inscura 서비스 주소나 토큰이 변경된 경우, Jellyfin 플러그인 설정에서 동기화하여 업데이트한 후 메타데이터를 새로 고쳐야 합니다.
- Inscura 미디어 라이브러리가 잠겨 있거나 닫혀 있는 경우, Jellyfin은 메타데이터를 읽을 수 없습니다.
- YouTube가 아닌 예고편은 플러그인을 통해 가져오지 말고, Inscura 앱의 NFO 기능을 통해 Jellyfin으로 내보내 스캔하십시오.

## 문제 해결

### Jellyfin에서 Inscura 플러그인이 표시되지 않는 경우

1. 저장소를 통해 설치한 경우, Jellyfin 서버에서 저장소 URL에 접근할 수 있는지 확인하십시오.
2. 수동으로 설치한 경우, 플러그인 파일이 Jellyfin 데이터 디렉터리 내의 `plugins/Inscura_[버전]/`에 있는지 확인하십시오.
3. 해당 디렉터리에 `Jellyfin.Plugin.Inscura.dll`이 있는지 확인하십시오.
4. Jellyfin 프로세스가 해당 파일을 읽을 수 있는 권한이 있는지 확인하십시오.
5. Jellyfin을 재시작하십시오.
6. Jellyfin 웹 버전을 다시 열고, 콘솔의 플러그인 페이지로 이동하여 확인하십시오.

### Jellyfin에서 플러그인은 표시되지만 메타데이터가 없는 경우

1. Jellyfin 서버가 설치된 기기에서 Inscura 상태 확인 주소에 접속하십시오.
2. Inscura 미디어 라이브러리가 열려 있고, 로컬 인터페이스 서비스가 실행 중인지 확인하십시오.
3. 플러그인 내의 Inscura API URL이 잘못된 `127.0.0.1`이 아닌지 확인하십시오.
4. 인터페이스가 토큰 인증을 사용하는 경우, 플러그인에 올바른 토큰이 입력되었는지 확인하십시오.
5. 영화 라이브러리의 메타데이터 다운로더에서 `Inscura`이 활성화되어 있는지 확인하십시오.
6. 영화에 대해 메타데이터 새로 고침을 실행하고, 새로 고침 모드에서 ‘모든 메타데이터 덮어쓰기’를 선택하십시오.

### 커버, 배경 이미지 또는 배우 프로필 사진이 표시되지 않는 경우

1. Jellyfin 서버가 Inscura API URL에 접근할 수 있는지 확인하십시오.
2. API가 토큰 인증을 사용하는 경우, 플러그인의 토큰이 올바른지 확인하십시오.
3. Inscura에서 해당 미디어나 배우에 대해 사용 가능한 이미지 리소스가 실제로 있는지 확인하십시오.
4. 영화 라이브러리의 이미지 다운로더에서 `Inscura`이 활성화되어 있는지 확인하십시오.
5. 해당 영화의 메타데이터를 새로 고치고, 필요한 경우 기존 이미지를 대체하도록 선택하십시오.

### 예고편이 가져오지 않음

1. 플러그인 설정에서 `Import YouTube trailers`이 활성화되어 있는지 확인하십시오.
2. Inscura에서 해당 예고편의 주소가 YouTube 주소인지 확인하십시오.
3. 현재 플러그인은 YouTube에 호스팅되지 않거나 온라인에 있는 예고편은 가져오지 않습니다.
4. YouTube 외부의 예고편을 가져오려면, Inscura 앱의 NFO 기능을 통해 내보낸 후 Jellyfin에서 스캔하도록 하십시오.

## 플러그인 업그레이드

### 플러그인 저장소를 통한 업그레이드

1. 플러그인 저장소 URL에 여전히 접속할 수 있는지 확인하십시오.
2. Jellyfin 콘솔의 플러그인 페이지로 이동하십시오.
3. `Inscura`에 사용 가능한 업데이트가 있는지 확인하십시오.
4. 업데이트를 설치하십시오.
5. Jellyfin을 재시작하십시오.

### 수동 업그레이드

1. Jellyfin을 중지합니다.
2. 새 버전의 zip 파일을 다운로드합니다.
3. `plugins` 디렉터리 아래에 새 버전 디렉터리(예: `Inscura_0.1.1.0`)를 생성합니다.
4. 새 버전의 zip 파일을 해당 디렉터리에 압축을 풉니다.
5. 새 버전이 정상적으로 로드되는지 확인할 때까지 이전 버전 디렉터리를 그대로 둡니다.
6. Jellyfin을 시작합니다.
7. 새 버전이 정상적으로 작동하는 것을 확인한 후, 이전 버전 디렉터리를 삭제합니다.
